# 智能健康膳食管理系统前后端接口契约

## 1. 契约说明

本文档面向 Vue3 前端与 Spring Boot 业务后端联调，基于《智能健康膳食管理系统业务后端设计文档-H2版》整理接口路径、请求参数、响应结构、枚举值和错误约定。

后端基础地址：

```text
http://localhost:8080
```

接口统一前缀：

```text
/api/v1
```

前端开发地址：

```text
http://localhost:5173
```

## 2. 全局约定

### 2.1 认证方式

登录成功后，后端返回 JWT。除游客接口外，前端请求头必须携带：

```http
Authorization: Bearer <token>
```

Token 过期或无效时，后端返回 `40100`，前端清空登录状态并跳转登录页。

### 2.2 统一响应

所有接口默认返回统一结构：

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "timestamp": "2026-06-24T12:00:00+08:00"
}
```

### 2.3 分页请求

分页接口统一使用 query 参数：

| 参数        | 类型   | 必填 | 默认值 | 说明            |
| ----------- | ------ | ---- | ------ | --------------- |
| `page`    | int    | 否   | 1      | 页码，从 1 开始 |
| `size`    | int    | 否   | 10     | 每页条数        |
| `keyword` | string | 否   | -      | 搜索关键词      |

分页响应统一结构：

```json
{
  "items": [],
  "page": 1,
  "size": 10,
  "total": 100,
  "pages": 10
}
```

### 2.4 日期时间格式

| 类型     | 格式           | 示例                          |
| -------- | -------------- | ----------------------------- |
| 日期     | `YYYY-MM-DD` | `2026-06-24`                |
| 日期时间 | ISO 8601       | `2026-06-24T12:00:00+08:00` |

### 2.5 字段命名

请求和响应 JSON 字段统一使用 camelCase，例如：

```json
{
  "dailyCalorieTarget": 1800,
  "targetWeightKg": 60.5
}
```

### 2.6 错误码

| 业务码    | HTTP 状态 | 含义                |
| --------- | --------- | ------------------- |
| `0`     | 200       | 成功                |
| `40001` | 400       | 参数校验失败        |
| `40100` | 401       | 未登录或 Token 失效 |
| `40300` | 403       | 权限不足            |
| `40400` | 404       | 资源不存在          |
| `40900` | 409       | 资源冲突            |
| `42200` | 422       | 业务规则校验失败    |
| `50200` | 502       | 外部 AI 服务不可用  |
| `50201` | 502       | AI 返回格式错误     |
| `50000` | 500       | 系统异常            |

参数校验失败示例：

```json
{
  "code": 40001,
  "message": "参数校验失败",
  "data": {
    "fieldErrors": [
      {
        "field": "heightCm",
        "message": "身高必须大于 0"
      }
    ]
  },
  "timestamp": "2026-06-24T12:00:00+08:00"
}
```

## 3. 枚举约定

| 枚举                            | 值                                                                                                |
| ------------------------------- | ------------------------------------------------------------------------------------------------- |
| 用户角色`role`                | `USER`, `ADMIN`                                                                               |
| 用户状态`userStatus`          | `ENABLED`, `DISABLED`                                                                         |
| 性别`gender`                  | `MALE`, `FEMALE`, `UNKNOWN`                                                                 |
| 活动水平`activityLevel`       | `SEDENTARY`, `LIGHT`, `MODERATE`, `HIGH`                                                  |
| 健康目标`healthGoal`          | `FAT_LOSS`, `MUSCLE_GAIN`, `SUGAR_CONTROL`, `MAINTAIN`                                    |
| 目标周期状态`goalCycleStatus` | `ACTIVE`, `COMPLETED`, `CANCELLED`, `EXPIRED`                                             |
| 餐次`mealType`                | `BREAKFAST`, `LUNCH`, `DINNER`, `SNACK`                                                   |
| 菜谱难度`difficulty`          | `EASY`, `NORMAL`, `HARD`                                                                    |
| 菜谱状态`recipeStatus`        | `DRAFT`, `PENDING`, `ONLINE`, `OFFLINE`                                                   |
| 膳食计划状态`mealPlanStatus`  | `ACTIVE`, `ARCHIVED`                                                                          |
| 膳食来源`sourceType`          | `MANUAL`, `RECIPE`, `PLAN`, `AI_RECIPE`                                                   |
| 计划反馈状态`feedbackStatus`  | `COMPLETED`, `PARTIAL`, `SKIPPED`, `REPLACED`                                             |
| 替换原因`replaceReason`       | `DISLIKE`, `NO_INGREDIENT`, `CALORIE_TOO_HIGH`, `CALORIE_TOO_LOW`, `NO_TIME`, `OTHER` |
| AI 菜谱状态`aiRecipeStatus`   | `GENERATED`, `CONFIRMED`, `SAVED`                                                           |
| 帖子状态`postStatus`          | `PENDING`, `ONLINE`, `REJECTED`, `OFFLINE`                                                |
| 风险等级`riskSeverity`        | `INFO`, `WARNING`, `DANGER`                                                                 |
| 风险指标`riskNutrient`        | `CALORIE`, `PROTEIN`, `FAT`, `CARBOHYDRATE`, `SODIUM`                                   |
| 风险规则操作符`riskOperator`  | `GT`, `LT`, `BETWEEN`                                                                       |
| 报告类型`reportType`          | `WEEKLY`, `MONTHLY`                                                                           |

## 4. 认证接口

### 4.1 注册

```http
POST /api/v1/auth/register
```

请求：

```json
{
  "username": "alice",
  "email": "alice@example.com",
  "password": "user123"
}
```

响应 `data`：

```json
{
  "id": 2,
  "username": "alice",
  "email": "alice@example.com",
  "role": "USER",
  "status": "ENABLED"
}
```

### 4.2 登录

```http
POST /api/v1/auth/login
```

请求：

```json
{
  "username": "alice",
  "password": "user123"
}
```

响应 `data`：

```json
{
  "token": "jwt-token",
  "tokenType": "Bearer",
  "expiresIn": 7200,
  "user": {
    "id": 2,
    "username": "alice",
    "email": "alice@example.com",
    "role": "USER"
  }
}
```

### 4.3 当前用户

```http
GET /api/v1/auth/me
```

响应 `data` 同登录响应中的 `user`。

### 4.4 修改密码

```http
PUT /api/v1/auth/password
```

请求：

```json
{
  "oldPassword": "user123",
  "newPassword": "newUser123"
}
```

响应 `data`：

```json
true
```

### 15.2 发布帖子

```http

Content-Type: application/json
```

权限：用户。
请求：

校验规则：
- `imageIds` 必填，数量 `1-9`。
- 只能引用当前用户本人上传且状态为 `TEMP` 的图片。
- 帖子相册顺序按 `imageIds` 顺序保存。
- 历史旧帖允许无图读取，但新发帖必须有图。

## 17. 前端联调建议

- 前端统一封装 `request`，自动注入 `Authorization`。
- 前端统一判断 `code !== 0` 的业务错误，不只依赖 HTTP 状态。
- 表单枚举选项直接使用本文档第 3 节。
- 列表页统一适配分页结构 `items/page/size/total/pages`。
- 图片上传接口使用 `multipart/form-data`，字段名固定为 `file`。
- AI 解析、营养报告生成、风险规则执行属于可能耗时接口，前端需要加载态和失败提示。
